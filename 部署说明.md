# 买菜清单 — Cloudflare 部署说明

这套东西跟你部署 SixFlamingos 的流程几乎一样，只是多了两步：建一个 KV 存数据、设一个口令。

## 项目结构

```
grocery-app/
├── index.html              ← 前端网页（带口令门和云同步）
└── functions/
    └── api/
        └── lists.js        ← 后端 API（Cloudflare 自动识别这个目录）
```

`functions/` 这个目录名是 Cloudflare 的约定——放在里面的代码会自动变成后端接口，
你不用单独配置。`functions/api/lists.js` 就对应网址 `/api/lists`。

---

## 部署步骤

### 第 1 步：把项目放上 Cloudflare Pages

跟 SixFlamingos 一样，两种方式任选：

**方式 A — 连 Git 仓库（推荐，跟你现有流程一致）**
1. 把 `grocery-app` 整个文件夹推到一个 GitHub 仓库
2. Cloudflare 控制台 → Workers & Pages → Create → Pages → Connect to Git
3. 选这个仓库。构建设置：
   - Framework preset: `None`
   - Build command: 留空
   - Build output directory: `/`（根目录，因为 index.html 就在根上）
4. Save and Deploy

**方式 B — 直接拖文件上传**
1. Workers & Pages → Create → Pages → Upload assets
2. 把 `grocery-app` 文件夹里的内容拖进去（注意要包含 `functions` 子目录）
3. Deploy

部署完会给你一个 `xxx.pages.dev` 的网址。**但现在还不能用**，因为 KV 和口令还没配——往下走。

### 第 2 步：建 KV 命名空间（数据存这儿）

1. Cloudflare 控制台 → Storage & Databases → KV → Create a namespace
2. 名字随便起，比如 `grocery-data`
3. 建好就行，不用往里填东西

### 第 3 步：把 KV 绑定到你的 Pages 项目

1. 回到你的 Pages 项目 → Settings → Functions（或 Bindings）→ KV namespace bindings
2. Add binding：
   - **Variable name** 必须填 `GROCERY`（大写，跟代码里对应，别改）
   - **KV namespace** 选你刚建的 `grocery-data`
3. 保存

### 第 4 步：设置口令

1. 同一个项目 → Settings → Environment variables（环境变量）
2. Add variable：
   - **Variable name** 填 `APP_PASSCODE`（别改这个名字）
   - **Value** 填你想要的口令，比如 `liude2026`（自己定，别太简单）
   - 建议点 “Encrypt”（加密）把它存成密钥
3. 保存

### 第 5 步：重新部署一次

改完绑定和变量后，要让它生效：
- Deployments → 最近一次部署 → 右边菜单 → Retry deployment
- 或者随便推一次代码触发新部署

---

## 用法

打开 `xxx.pages.dev` → 输入第 4 步设的口令 → 进入。
你、家人、不管在美国还是中国，输同一个口令，看到的就是同一份清单。
谁改了，对方刷新一下（或下次打开）就能看到最新的。

口令在你这次浏览器会话里会记住，关掉标签页再开要重输（手机上加到主屏幕后用起来跟 App 差不多）。

---

## 几个要知道的点

**关于“同步”到底怎么发生的**
每次你改清单，网页会在你停手约 0.8 秒后自动把整份数据上传到云端（右上角小圆点会变黄“保存中”→绿“已同步”）。别人打开时拉的就是云端最新的。它**不是实时多人协作**——如果你和家人同一秒都在改同一份，后保存的会覆盖先保存的。对你们“一个人列单、另一个人照着买”的场景完全够用。想强制同步可以点“↻ 立即同步”。

**关于口令的安全级别**
这是个轻量口令，挡住路人足够了，但别拿它存敏感信息（它本来也就是买菜单）。口令是明文比对，走的是 HTTPS 加密传输，正常用没问题。想换口令就改第 4 步的 `APP_PASSCODE` 再重新部署。

**关于中国可达性**
跟 SixFlamingos 一样走 Cloudflare 全球网络，数据接口也在同一张网上，所以网页能开的地方数据通常也能存取。但如前所说，境外 CDN 在中国是“通常可以”而非“绝对保证”，这点不备案无法根除。如果哪天中国那边打不开，多半是网络层面的临时波动，不是你配置错了。

**免费额度**
Cloudflare 免费层：KV 每天 10 万次读、1000 次写，Pages Functions 每天 10 万次请求。你们买菜这点用量，连零头都用不到。

## 出问题时怎么查

- **输对口令也进不去 / 提示“连接失败”**：多半是 KV 没绑定或变量名拼错。检查第 3 步变量名是不是 `GROCERY`、第 4 步是不是 `APP_PASSCODE`，改完记得重新部署（第 5 步）。
- **提示“口令不对”但你确定没输错**：确认 `APP_PASSCODE` 的值有没有多余空格；改过变量后是否重新部署了。
- **网页能开但一直“保存失败”**：通常是 KV 绑定丢了，重做第 3 步并重新部署。
